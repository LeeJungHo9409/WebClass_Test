# GIT
- .git 폴더가 위치한 폴더의 모든 내용을 관리해주는 프로그램
- .git 폴더가 위치한 폴더를 Repository라고 부른다
- .git 폴더 내부에는 해당 폴더의 모든 정보가 기록되어 있다.
- .git 폴더 내부에 저장된 기록을 활용하여 언제든지 이전 상태로 돌아갈 수 있다.
- GIT 은 관리하는 폴더 내부의 무언가가 아주 약간만 변해도 모두 알아채고 관리해준다.
- Eclipse 에는 Egit이 내장되어 있다.

# Commit
 - 레포지토리 내부의 변경 사항을 확정짓는 명령어
 - 커밋한 시점은 복구 지점이 된다. (Check out 대상이 된다)
 - 체크 아웃을 통해 언제든지 원하는 커밋으로 HEAD를 옮겨다닐 수 있다.
 	* HEAD는 현재 깃이 보여주고 있는 상태

# branch
  - 하나의 레포지토리를 다양한 형태로 보관할 수 있는 기능
  - 다른 브랜치로 변경하는 것도 체크 아웃이라고 한다.
  - 커밋하지 않는 상태로 다른 브랜치로 체크아우샇면 모든 작업내용이 사라진다.
  
# Merge
  - 한 브랜치의 HEAD로 다른 브랜치의 HEAD를 합치는 것
  - 합칠 때 같은 파일의 같은 부분을 수정했다면, 둘중 하나를 선택하는 심사과정을 거쳐야 한다 (충돌)
